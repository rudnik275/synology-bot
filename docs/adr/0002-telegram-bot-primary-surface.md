# Telegram Bot as primary surface; Mini App / PWA deferred

Рассматривались пять вариантов поверхности: чистый Telegram Bot, Telegram Mini App, отдельный PWA с собственной auth, VPN+DSM UI, нативное приложение. Выбран **чистый Bot**, потому что он бесплатно даёт пять штук, за которые любая альтернатива должна платить инженерией: auth по username, push-уведомления, отсутствие требования к публичному IP, передача файлов через нативный Telegram-UI, «уже открыт в кармане». Ограничения UI (только текст + кнопки) приемлемы для single-user low-frequency сценария.

Mini App остаётся опцией — может быть добавлен поверх существующего бота точечно (rich UI для дашборда / результатов поиска), если конкретные места UI станут реально болеть. Архитектурное следствие: домен-логика должна жить отдельно от Telegram-handler'ов, чтобы Mini App потом было дёшево прикрутить.
