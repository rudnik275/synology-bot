# Domain boundary — in-flight downloads, not a library

«Удобный трек файлов с торрента» был неоднозначен: (a) отслеживать **активные** скачивания (статус, %, ошибки, завершение) или (b) отслеживать **библиотеку** на NAS (что лежит, какие эпизоды есть). Выбрано **(a)**. Как только файл лёг в `/video/<папку>` и Synology перевёл задачу в финальный статус — она выпадает из домена бота. Что происходит с файлом дальше (Plex, Jellyfin, ручной просмотр, переименование, организация) — не наша забота.

Аргумент: библиотечный домен покрывается Plex/Jellyfin/Synology File Station, бот там будет хуже специализированных инструментов; phone-in-pocket value prop — «следить за активным циклом и реагировать на события», не «листать инвентарь». NAS Health-функции (диск, температура) **не** нарушают эту границу — они про NAS как машину, не про контент на нём.
