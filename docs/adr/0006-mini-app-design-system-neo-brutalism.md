# Mini App design system — Neo-Brutalism, single light mode, jobs-first IA

Mini App из [[0005]] нужен визуальный язык и информационная архитектура. Прогнали продуктовые цели (пульт к NAS, owner-only, часто открывается ИЗ пуша) через UX-skill и сравнили 5 отрендеренных вариантов одного экрана (neo-brutalism / dark-ops-terminal / iOS-native / flat-bright / glass-cinematic). **Решение: Neo-Brutalism** — переиспользуем дизайн-язык pet-проекта voice-clip (кремовый `#FFFDF5`, чёрный `#09090B`, бордеры 3/5px, жёсткие offset-тени `Npx Npx 0` без блюра, повёрнутые стикеры-бейджи, dot-grid фон, Space Grotesk, насыщенные акценты red/violet/yellow/green/orange). Берём именно стиль и токены, не код. Это даёт «фирменный характер» и одну семью с voice-clip ценой того, что инструмент намеренно НЕ выглядит «нативным Telegram» (native/flat варианты отвергнуты как безликие; dark-ops был фаворитом «по домену», но проиграл характеру; glass отвергнут из-за риска `backdrop-filter`-джанка в Telegram-webview).

**Один режим, не тема.** Сознательно отказываемся от light/dark-toggle и от посадки на `--tg-theme-*` (разворот текущего `frontend/src/style.css`): кремовый фон — это и есть идентичность neo-brutalism, инверсия её ломает. Приложение всегда светлое, тему пользователя Telegram игнорируем (отклонение от идиома `platform-adaptive` — осознанное, ради узнаваемости).

**IA — от jobs-to-be-done, а не от исходного плана 4 табов.** Три bottom-таба **Downloads / NAS / Shows**, дефолт — **Downloads** (а не NAS из исходного плана: NAS-health — push-driven назначение, не browse-surface). Главное действие приложения — «добавить закачку» — поднято в **единый Add-флоу** (FAB → лист с тремя режимами: Search / magnet / .torrent → folder-picker → go); поиск перестаёт быть отдельным равноправным табом, т.к. это лишь один из трёх способов добавления. Здоровье NAS обслуживается амбиентно **чипом в хедере** (green/amber/red точка + одна метрика, тап → NAS-таб) на каждом экране. Deep-link из пушей открывает контекст: finished/stuck/failed → Downloads (на задачу); disk/temp/offline → NAS; серия вышла → Shows. Следствия для слайсов [[0005]]: Search (#62) и Add task (#63) сливаются в один Add-флоу; #60/#61 переименовывают табы и меняют дефолт; #65 получает deep-link-карту.

**Анимации — отдельная ось работы, не «доделаем потом».** В voice-clip моушн несёт половину характера: механический press (элемент «вдавливается» в свою тень — `translate(N,N)` + тень схлопывается в `0 0 0`), spring-поп модалок (`cubic-bezier(0.34,1.4,0.64,1)`), стикеры с поворотом, stagger списков. Статичный neo-brutalism выглядит дёшево — поэтому моушн закладываем как first-class требование с самого начала, не как полиш в конце.

**Реализация — без анимационной библиотеки.** Весь нужный моушн (press, слайд+поп Add-листа, stagger списка закачек, ширина прогресс-баров, пульс статуса) достижим чистым CSS (`transition`/`@keyframes`/`cubic-bezier`) + встроенным Vue `<Transition>`/`<TransitionGroup>` (последний бесплатно даёт FLIP-перестановку списков) — ровно как в voice-clip, где анимационных либ ноль. Нео-брутализм-моушн намеренно **механический, не пружинно-органический** (в voice-clip press = `100ms linear`, единственная «пружина» — поп модалки `cubic-bezier(0.34,1.4,0.64,1)`), поэтому **анимационную библиотеку не тянем вовсе** — GSAP/Motion и пр. это over-engineering для приложения-из-карточек (их сила — сложные timeline/scroll/SVG-morph/FLIP, которых здесь нет). Если однажды вылезет реально сложная сцена — решим точечно тогда.

**Грамотность важнее реализации** (приоритет владельца): «где какая анимация уместна» берём из принципов `ui-ux-pro-max` Animation-домена (длительности 150–300ms, ease-out-вход/ease-in-выход, exit короче enter, модалки из точки-источника, stagger 30–50ms, motion-meaning) + планируемого проектного motion-скилла под наш нео-брутализм/Vue. Обязательное: уважать `prefers-reduced-motion`, держать 60fps в Telegram-webview на слабых телефонах.
