# Owner-only single-user bot

Бот обслуживает один личный Synology NAS одного владельца. Доступ проверяется примитивно: Telegram `chat.id` сверяется с конфигурируемым числовым `OWNER_CHAT_ID`, остальные получают отказ. Многопользовательский режим (включая «approved family list») отброшен сознательно — он тянет за собой namespacing подписок/настроек/задач, права на чужие задачи и UI для управления списком пользователей. Use case реально однопользовательский, и единственный JSON подписок без `userId` — намеренное упрощение, не недоработка.

Раньше сверка была по `chat.username` против `OWNER_USERNAME`. Сменили на `chat.id`, потому что (1) numeric ID стабилен, username юзер может поменять или удалить; (2) у Telegram-пользователя может вообще не быть username, и тогда сверка молча проваливалась; (3) тот же `OWNER_CHAT_ID` переиспользуется compose-файлом для сборки Watchtower notification URL — одно значение в .env вместо двух идентификаторов владельца + дублирующегося BOT_TOKEN.
