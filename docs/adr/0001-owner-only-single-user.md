# Owner-only single-user bot

Бот обслуживает один личный Synology NAS одного владельца. Доступ проверяется примитивно: Telegram `chat.username` сверяется с конфигурируемым `OWNER_USERNAME`, остальные получают отказ. Многопользовательский режим (включая «approved family list») отброшен сознательно — он тянет за собой namespacing подписок/настроек/задач, права на чужие задачи и UI для управления списком пользователей. Use case реально однопользовательский, и единственный JSON подписок без `userId` — намеренное упрощение, не недоработка.
